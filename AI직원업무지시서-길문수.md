## 2. 만들 AI 직원

PDF 건축물대장·등기부등본에서 핵심 정보를 자동 추출하고, 네이버 블로그 광고글 형식으로 작성해주는 AI 직원

---

## 3. 반복 업무와 해결할 문제

**핵심 문제**: 직원이 매달 100개 광고글을 네이버 블로그에 올릴 때 정보 입력 실수와 시간 소요 발생

**자동화 후 기대 효과**: 직원 100개/월 반복 작업 자동화로 시간 및 오류 단축

**대상 사용자**: 길문수 — 부동산·건설 업무 담당자

---

## 4. 입력값과 처리 흐름

Claude 또는 ChatGPT + PDF 읽기 기능, 현재 사용 중인 네이버 블로그 광고글 예시 2개, 건축물대장/등기부등본 샘플 PDF 1개

---

## 5. 오늘 만들 MVP

**MVP 범위**: PDF 업로드 → 핵심 정보 자동 추출 → 네이버 블로그 글 형식으로 자동 작성까지 (수동 복사-붙여넣기 가능한 상태)

오늘은 위 범위 안에서 실제로 한 번 실행되는 결과물을 만드는 것이 목표입니다.
기능을 넓히기보다, 신청자가 직접 넣은 입력값 1개가 처리되어 결과 1개로 나오는 흐름을 먼저 완성합니다.

---

## 6. 첫 화면에 필요한 최소 요소

- 입력 영역 (텍스트 입력 또는 파일 업로드)
- 실행 버튼
- 결과 출력 영역
- 에러가 났을 때 다시 시도할 수 있는 안내

---

## 7. 오늘 만들지 않는 것

수업 당일 결과물을 내기 위해 아래는 Phase 2 이후로 미룹니다.

- 음성 변환 / TTS 기능
- 생성 결과 저장 / 이력 관리
- 결과물 공유 · 이메일 전송
- 다중 파일 동시 처리
- 로그인 / 회원가입 / 결제
- 외부 배포 링크 생성

---

## 8. 개발 진행 기준

**Phase 1 — 오늘 MVP**
- 위 5번 MVP 범위만 구현
- 실제 샘플 입력 1개로 결과가 화면에 나오는지 확인
- 신청자가 결과물을 복사하거나 바로 사용할 수 있게 마무리

**Phase 2 이후 — 수업 후 확장**
- 저장 / 이력 관리
- 공유 / 이메일 전송
- 로그인 / 결제
- 외부 배포
- 다중 입력 처리

**항상 주의**
- API 키는 코드에 직접 쓰지 않기 — 환경변수(.env)로 관리
- 개인정보는 샘플 데이터로 대체해서 테스트
- GitHub Public 저장소에 .env 파일 올리지 않기

---

## 9. 성공 기준

만든 AI 직원으로 실제 PDF 5개를 테스트해서 직원이 블로그에 올릴 수 있는 글이 나오는지, 오류율이 실제로 줄었는지 확인

**수업 당일 검증 방법**: 실제 입력값 1개를 넣어 결과가 화면에 나오는지 직접 확인
